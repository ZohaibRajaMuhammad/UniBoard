# Responsive Layout Standards

## Breakpoints
- Large desktop: 1440px and above
- Desktop/laptop: 1280px to 1439px
- Small laptop/tablet landscape: 1024px to 1279px
- Tablet portrait: 768px to 1023px
- Mobile: below 768px

## Layout Rules
- Desktop uses a persistent left sidebar.
- Tablet begins collapsing low-priority rails and reducing grid breadth.
- Mobile removes the sidebar and switches to floating bottom navigation.
- Every page must maintain first-action visibility at every breakpoint.

## Spacing
- Outer content padding desktop: 28px x 32px
- Tablet content padding: 24px x 24px
- Mobile content padding: 16px to 20px
- Vertical section rhythm: 20px to 32px depending on page importance

## Width Logic
- Content pages: 760px to 980px main zones
- Analytical or discovery pages: up to 1100px wide
- Planner and reputation use split layouts with a right rail

## Mobile Behavior
- Use stacking instead of squeezing
- Collapse side panels into drawers or bottom sheets
- Preserve large touch targets
- Make AI entry points thumb reachable
- Keep key CTAs visible without excessive scrolling

## Scaling Standard
- No page should require horizontal scrolling
- No key card should clip text at mobile widths
- Cards must reflow before text compresses too far
- Tables or dense metric rows should become stacked modules on smaller devices
