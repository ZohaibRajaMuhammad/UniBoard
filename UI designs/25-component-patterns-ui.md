# Component Patterns UI

## Priority Components
- glass card
- room card
- post card
- comment thread
- composer
- search bar
- modal shell
- toast system
- leaderboard row
- calendar grid

## Shared Visual Rules
- Consistent border logic
- Consistent radius language
- Consistent spacing rhythm
- Consistent metadata dimming
- Consistent action hierarchy

## Post Card Pattern
- strong author/type row
- readable body text
- compact metadata
- clean action row
- optional AI helper row

## Analytical Card Pattern
- concise title
- clear metric or chart
- minimal supporting text
- optional AI insight block

## Form Pattern
- explicit labels
- strong focus rings
- clear helper/error text
- no ambiguous state changes

## Reuse Rule
- Pages should compose from shared patterns, not invent new local variants unless the page has a strong product reason.
