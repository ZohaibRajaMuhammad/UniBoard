# UniBoard Execution System

This package organizes the entire `docs` repository into a professional delivery system so you can complete every documented requirement in a controlled order.

## Goal
Use the already-written documentation in `docs` as the source of truth, then execute the product in a repeatable 4-step workflow across 5 operational folders.

## Folder Structure
1. `01-master-requirements`
2. `02-planning-and-mapping`
3. `03-build-workflow`
4. `04-validation-and-qa`
5. `05-release-and-closure`

## The 4-Step Workflow Used In Every Folder
1. Read the relevant source READMEs from the main `docs` folders.
2. Convert them into actionable implementation or review tasks.
3. Execute and verify the dependencies for that phase.
4. Sign off the phase before moving to the next folder.

## Coverage
This execution system is designed to cover:
- `/docs/screens`
- `/docs/features`
- `/docs/ai-features`
- `/docs/components`
- `/docs/flows`
- `/docs/api`
- `/docs/states`
- `/docs/validations`
- `/docs/product-package`
- `/docs/tasks`

## Usage Rule
Do not skip folders. Each folder exists to prevent incomplete implementation of the written READMEs.
