# Tasks Documentation Index

This folder contains delivery-oriented work planning derived from the UniBoard documentation system.

## Files In This Folder
- `implementation-backlog.md`

## How To Use These Files
1. Use the backlog after the requirement and planning phases are complete.
2. Map backlog items to the related screen, feature, flow, API, state, and validation files.
3. Track implementation progress against documented completion standards.

## Completion Standard
Task planning is complete only when every backlog item points back to a documented requirement and no undocumented implementation work is driving the schedule.
