## How To Use These Files
1. Read the feature README to understand why the API exists.
2. Read the API README for contract expectations and error handling.
3. Use the OpenAPI contract for typed implementation and QA.
4. Verify each endpoint against the flows and validations before signoff.

## Completion Standard
An API area is complete only when request shapes, responses, authorization, retries, errors, and observability match both the API docs and the OpenAPI contract.
